32/** This concrete class implements the Red-Green-Blue LED interface by sending
33 the LED state over the TPipe to an external client. The driver assumes the
34 following TPipe command format (without leading/trailing SOF/EOF framing characters)
35 \code
36
37 <DD> <HH:MM:SS.sss> writeRGBLed <led> <r> <g> <b> <bright>
38 Where:
39 <DD> is CPU time since power-up/reset: Format is: DD HH:MM:SS.sss
40 <HH:MM:SS.sss> is CPU time since power-up/reset: Format is: DD HH:MM:SS.sss
41 <led> LED identifier/label (with no whitespace allowed)
42 <r> The 'Red' value (0-255) for the RGB LED
43 <g> The 'Green' value (0-255) for the RGB LED
44 <b> The 'Blue' value (0-255) for the RGB LED
45 <bright> Overall brightness for the LED (0-255)
46
47 \endcode
48 */
